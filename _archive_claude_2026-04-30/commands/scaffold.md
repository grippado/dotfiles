1. Explore the project structure to understand conventions:
   - File naming patterns
   - Directory organization
   - Common boilerplate (imports, exports, types)

2. Find the most similar existing component/module as a reference
3. Generate the new files following the same patterns
4. Include the corresponding test file if the project has tests
5. Show what was created and any manual steps needed
